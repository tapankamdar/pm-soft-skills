# PM Planning & Strategy

This skill covers the planning and strategy work PMs do each quarter: setting goals that mean something, building a vision that travels, making hard prioritization calls, and crafting the narrative that makes people want to build.

## Core principle

All PMs make plans. Great PMs make plans that create direction — so that every feature decision, every tradeoff, every sprint becomes obvious in relation to it.

> "A roadmap with 12 priorities has no priorities. Prioritization is not a framework problem. It's a courage problem."

## Before responding — gather context with AskUserQuestion

Before diving in, ask the following questions using AskUserQuestion. Ask all questions in a single call (no more than 3).

**Question 1 — What planning task are you working on?**
Options: `Setting quarterly goals or OKRs`, `Building a product vision or strategy`, `Prioritizing the roadmap or saying no`, `Crafting a narrative or pitch`, `Fixing a broken process or system on my team`

**Question 2 — Where are you in the process?**
Options: `Starting from scratch`, `Refining something I already have`, `Preparing for a review or presentation`, `Dealing with too many competing priorities`

**Question 3 — What's the hardest part right now?**
Options: `Getting alignment — people aren't bought in`, `Clarity — I don't know what to cut`, `Confidence — I'm not sure my vision is right`, `Urgency — I need this done quickly`

Use the answers to select the right reference file and frame the response at the right level.

## How to help

1. Ask the 3 context questions above using AskUserQuestion.
2. Read the reference file that matches the PM's task.
3. Apply the specific framework — PACE formula, the Sell Tour, the tradeoff table, or the Future Tweet test.
4. If the PM shares a draft vision, goals, or roadmap, evaluate it against the framework and give specific feedback.
