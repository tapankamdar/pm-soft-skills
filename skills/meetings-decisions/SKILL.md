# PM Meetings & Decisions

This skill covers the decisions and meetings at the center of a PM's week: running meetings that actually produce outcomes, aligning stakeholders before the room gathers, facilitating teams toward decisions using questions instead of answers, and making product calls faster.

## Core principle

All PMs run meetings. Great PMs make decisions before the meeting starts — and use the meeting to confirm, not debate.

> "If your meeting is filled with debate, you're too late. Surprise kills trust. Silence kills support."

## Before responding — gather context with AskUserQuestion

Before diving in, ask the following questions using AskUserQuestion. Ask all questions in a single call (no more than 3).

**Question 1 — What situation are you preparing for?**
Options: `An upcoming meeting I need to run well`, `A decision I need to drive`, `Getting alignment before a big room`, `A team discussion that keeps going in circles`

**Question 2 — What's at stake?**
Options: `A major product or roadmap decision`, `Stakeholder or exec buy-in`, `Cross-functional alignment`, `Resolving a debate the team is stuck on`

**Question 3 — What's the current blocker?**
Options: `People debate in the room instead of deciding`, `I can't get the right people aligned beforehand`, `I'm not sure how to frame the decision`, `The team can't reach consensus`

Use the answers to select the right reference file and tailor the guidance to the specific situation.

## How to help

1. Ask the 3 context questions above using AskUserQuestion.
2. Read the reference file that matches the PM's situation.
3. Apply the specific framework — the confirmation-point approach, the pre-read strategy, the 30-minute decision process, or the three questions that reset a room.
4. If the PM describes a meeting that went poorly, diagnose which step was missing and give specific guidance.
