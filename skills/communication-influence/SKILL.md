# PM Communication & Influence

This skill covers the soft power of PM communication: adapting your message for different audiences, simplifying complex ideas so they land, selling belief not just information, using stories to drive alignment, and maintaining trust when things don't go to plan.

## Core principle

All PMs communicate. Great PMs influence. The difference is not what you say — it's how deeply you understand the person you're saying it to, and whether you give them a reason to act, not just a reason to understand.

> "You don't just ship products — you sell belief. You move hearts." — Tapan Kamdar

## Before responding — gather context with AskUserQuestion

Before diving in, ask the following questions using AskUserQuestion. Ask all questions in a single call (no more than 3).

**Question 1 — What are you trying to do?**
Options: `Adapt my message for a specific audience`, `Simplify something complex so it lands`, `Sell an idea or get buy-in`, `Rebuild trust after a miss or delay`

**Question 2 — Who is the audience?**
Options: `Engineering team`, `Executives or leadership`, `Cross-functional partners (design, marketing, data)`, `Skeptical stakeholder or decision-maker`

**Question 3 — What's the challenge?**
Options: `They don't engage or seem interested`, `They push back or don't believe it`, `They don't understand the problem`, `They've lost trust in me or my team`

Use the answers to select the right reference file and tailor the approach.

## How to help

1. Ask the 3 context questions above using AskUserQuestion.
2. Read the reference file that matches the PM's challenge.
3. Apply the specific framework — the translation test, the Problem-Path-Payoff structure, the Sell Tour, the user story structure, or the trust-reset framework.
4. If the PM shares something they've already communicated that didn't land, diagnose why and help reframe it.
